# MYNVOICE — Design System (Petrol & Coral)

## 1. Design Direction

This design direction defines the visual identity of MYNVOICE.

It aims to be:
- Modern but not trendy
- Distinct from typical SaaS (no blue/purple clichés)
- Clean, but with personality
- Professional with subtle warmth

This palette combines:
- Deep petrol tones (trust, stability)
- Coral accents (energy, action, human touch)

---

## 2. Core Color Palette

### Primary Colors

- Petrol Dark: `#0F4C5C`
- Petrol Mid: `#2C7A7B`

### Accent

- Coral: `#FF6B6B`

### Neutral Base

- Light Background: `#F0F3F5`
- Dark Surface: `#1B263B`

---

## 3. Semantic Usage

### Backgrounds
- Main background: `#F0F3F5`
- Cards / surfaces: `#FFFFFF`
- Dark sections: `#1B263B`

### Primary Actions
- Buttons: `#0F4C5C`
- Hover: `#2C7A7B`

### Secondary / Highlights
- Coral (`#FF6B6B`) for:
  - CTAs
  - Important actions
  - Notifications
  - Highlights

### Text
- Primary text: `#1B263B`
- Secondary text: `#5C677D`
- Inverse text: `#FFFFFF`

---

## 4. UI Behavior Guidelines

- Avoid overusing coral — it must feel intentional
- Petrol is the structural color
- Coral is the emotional/action color

### Examples:
- "Create Invoice" → Coral button
- Navigation → Petrol tones
- Background → Light neutral

---

## 5. Component Guidelines

### Buttons
- Primary: Petrol background + white text
- Secondary: Outline (Petrol border)
- CTA: Coral background

### Cards
- White background
- Soft shadow
- Rounded corners (12px–16px)

### Inputs
- Clean borders
- Subtle focus states using petrol tones

---

## 6. Data Visualization

Charts should:
- Use Petrol tones for base data
- Use Coral for highlights (e.g., current month, selected data)

Avoid rainbow charts.

---

## 7. UX Tone

The interface should feel:
- Fast
- Lightweight
- Calm
- Precise

Avoid:
- Heavy gradients
- Overly flashy UI
- Visual noise

---

## 8. Future Considerations

- Dark mode support:
  - Background: `#1B263B`
  - Cards: slightly lighter dark tones
  - Maintain coral for contrast

---

## 9. Logo Direction (Concept)

The logo should NOT be generic.

Avoid:
- Dollar signs
- Generic document icons
- Overused fintech symbols

Aim for:
- Minimal abstraction
- Geometric clarity
- Strong identity even in small sizes (favicon)
